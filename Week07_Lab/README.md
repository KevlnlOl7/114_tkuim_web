# KevinTkuIMWeb — Signup Form

## 自訂錯誤條件（Custom Validation）

| 欄位名稱 | 驗證條件 | 錯誤提示訊息 |
|-----------|-----------|----------------|
| 姓名 | 必填 | 請輸入姓名 |
| Email | 必填、格式需正確 | 請輸入正確的 Email |
| 手機 | 必填、10 碼數字、09 開頭 | 手機格式錯誤，請輸入 09 開頭的 10 碼數字 |
| 密碼 | 必填、至少 8 碼、需含大小寫與數字 | 請輸入密碼 |
| 確認密碼 | 必填、需與密碼一致 | 兩次密碼不一致 |
| 興趣標籤 | 至少選擇 1 個（含其他選項） | 請至少選擇一個興趣 |
| 條款勾選 | 必須閱讀至底才能勾選 | 請閱讀並同意條款才能繼續 |

## 驗證機制說明

### 🔹 即時驗證（on blur / on input）
- `blur`：欄位失焦時顯示錯誤訊息。
- `input`：輸入時即時清除錯誤與更新狀態。
- 密碼區同時更新強度條、文字提示與 checklist。

### 🔹 送出攔截（submit event）
- 綜合檢查所有欄位的有效性。
- 若有錯誤，自動聚焦第一個錯誤欄位。
- 條款未勾選會彈出提醒並中斷送出。
- 全部通過後模擬 1 秒送出狀態，顯示「註冊成功！」。

### 🔹 防重送（Anti double-submit）
- 送出中按鈕禁用並顯示「建立中...」。
- 完成後自動恢復按鈕狀態。

## 開發筆記

- 採用 **事件委派 (Event Delegation)** 處理興趣標籤，提升可維護性與擴充性。
- 所有錯誤訊息透過 `setCustomValidity()` 控制，並同步更新 `<p class="field-error">`。
- 密碼強度條使用 `score / 4 * 100%` 計算進度，顏色由紅 → 橘 → 綠變化。
- 「其他興趣」輸入框僅在 `data-value="other"` 啟用時顯示。
- 條款彈窗使用 `scroll` 事件判斷是否滾到底，控制勾選解鎖。
- 頁腳 Terms 鍵與主條款按鈕共用同一 `openTermsBtn.click()`。
- 小動物眼睛透過 CSS 變數 `--eye-x` / `--eye-y` 實現視線追蹤與撇頭動畫。
- 所有暫存資料透過 `localStorage` 保存與復原，避免意外刷新造成資料遺失。
